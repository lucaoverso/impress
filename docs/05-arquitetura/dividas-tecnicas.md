# Dividas tecnicas

Este documento lista dividas tecnicas priorizadas a partir do codigo atual e dos levantamentos de arquitetura, dados, fluxos e operacao. A lista nao propoe uma reescrita completa; ela serve como backlog de atencao tecnica.

## Criterios

| Nivel | Criterio usado |
| --- | --- |
| Critico | Pode afetar seguranca, continuidade operacional ou perda de dados. |
| Alto | Dificulta evolucao de modulos centrais ou aumenta muito o risco de regressao. |
| Medio | Afeta manutencao, clareza ou consistencia, mas com impacto mais localizado. |
| Baixo | Melhoria desejavel, com baixo risco imediato. |

## Observacoes

- As prioridades consideram risco tecnico e operacional, nao apenas tamanho de arquivo.
- Itens criticos nao implicam correcao imediata por codigo nesta etapa; eles indicam pontos que precisam de decisao antes de ampliar uso em producao.
- Qualquer correcao futura deve seguir `AGENTS.md`: escopo pequeno, um modulo por vez, sem alterar comportamento sem autorizacao.
