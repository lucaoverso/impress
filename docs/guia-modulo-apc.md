# Guia do Modulo APC

Este guia foi preparado para orientar professores e equipe gestora no uso do modulo APC, chamado na interface de `Central de Anexos`.

## 1. Objetivo do modulo

O APC foi criado para organizar solicitacoes de materiais pedagogicos e o envio dos arquivos pelos professores em datas definidas pela escola.

Com ele, a escola consegue:

- cadastrar solicitacoes de entrega por data;
- definir prazo para envio;
- escolher quem deve enviar o arquivo;
- acompanhar pendencias e envios realizados;
- consultar e baixar os anexos enviados.

## 2. Quem usa o modulo

O modulo pode aparecer de formas diferentes conforme o perfil do usuario:

- `Professor`: visualiza suas entregas e envia os arquivos solicitados.
- `Gestao`: cadastra solicitacoes, acompanha pendencias e consulta os arquivos recebidos.
- `Professor com acesso de gestao`: pode alternar entre `Minhas entregas` e `Gestao`.

## 3. Como acessar

1. Entre no sistema com seu usuario e senha.
2. Acesse a tela de `Servicos`.
3. Clique no cartao da `Central de Anexos`.
4. Escolha o `Ano letivo` no topo da tela, quando necessario.

## 4. Visao geral da tela

Na tela do APC, os elementos principais sao:

- `Calendario`: mostra os dias com entregas previstas.
- `Painel da data selecionada`: resume o que existe para o dia escolhido.
- `Minhas entregas`: area do professor para anexar e acompanhar arquivos.
- `Gestao`: area da equipe gestora para cadastrar e monitorar solicitacoes.

Se o usuario tiver os dois acessos, aparecerao as abas:

- `Minhas entregas`
- `Gestao`

## 5. Fluxo do professor

### 5.1 Selecionar a data

1. Use o calendario para escolher a data desejada.
2. Observe o resumo exibido no painel.
3. Abra a pendencia correspondente.

O sistema mostra apenas as entregas relacionadas ao professor logado.

### 5.2 Entender o que deve ser enviado

Cada entrega pode estar vinculada a:

- uma disciplina e turma especificas;
- uma entrega geral sem turma vinculada;
- uma data com prazo definido pela escola.

No cartao da entrega, o professor consegue ver:

- o nome da disciplina;
- a turma, quando existir;
- o status `Pendente`, `Enviado` ou `Prazo encerrado`;
- o prazo de envio;
- observacoes cadastradas pela gestao, quando houver.

### 5.3 Enviar um arquivo

1. Abra a entrega pendente.
2. Clique no campo `Arquivo`.
3. Escolha o arquivo no computador.
4. Clique em `Enviar arquivo`.

Depois do envio:

- o sistema registra a data e hora do envio;
- o status passa para `Enviado`;
- o arquivo fica disponivel para consulta e download.

### 5.4 Entender o nome exibido do arquivo

O APC agora mostra dois nomes quando necessario:

- `nome do usuario`: o nome do arquivo escolhido no computador;
- `nome do sistema`: o nome padronizado salvo pelo sistema.

Exemplo:

- arquivo escolhido pelo professor: `atividade_7A.pdf`
- nome salvo no sistema: `Atividade semanal - Professor Joao - 2035-05-14.pdf`

Isso ajuda o professor a reconhecer o arquivo enviado sem perder o padrao interno adotado pela escola.

### 5.5 Baixar um arquivo ja enviado

Quando a entrega ja estiver concluida:

1. abra o cartao da entrega;
2. clique em `Baixar arquivo`.

O arquivo sera baixado com o nome padronizado do sistema.

### 5.6 Remover e reenviar um arquivo

Se o prazo ainda estiver aberto, o professor pode:

1. abrir a entrega ja enviada;
2. clicar em `Remover arquivo`;
3. confirmar a remocao;
4. enviar uma nova versao.

Importante:

- a remocao so deve ser feita quando realmente for necessario substituir o anexo;
- depois da remocao, a entrega volta a ficar pendente ate o novo envio.

### 5.7 O que acontece quando o prazo encerra

Quando o prazo estiver encerrado:

- o professor ainda pode consultar o anexo enviado;
- o botao de remocao deixa de ficar disponivel;
- nao e possivel enviar uma nova versao.

Se houver necessidade de ajuste apos o encerramento, a orientacao e procurar a equipe gestora.

## 6. Fluxo da gestao

### 6.1 Alternar para a visao de gestao

Usuarios com permissao de acompanhamento devem:

1. abrir a `Central de Anexos`;
2. clicar na aba `Gestao`;
3. selecionar uma data no calendario.

Ao selecionar uma data, o formulario lateral de cadastro fica disponivel.

### 6.2 Cadastrar uma solicitacao de entrega

No card `Nova solicitacao de entrega`, preencha:

- `Data de referencia`
- `Prazo de envio`
- `Documento solicitado`
- `Quem deve enviar`
- `Observacao`

Depois:

1. clique em `Salvar solicitacao`;
2. confira o painel da data escolhida;
3. valide se os professores elegiveis apareceram corretamente.

### 6.3 Definir quem deve enviar

O campo `Quem deve enviar` controla o publico da solicitacao.

Em linhas gerais, a entrega pode ser direcionada para:

- todos os professores;
- apenas os professores relacionados ao horario daquele dia.

Se houver duvida sobre quem recebera a solicitacao, salve e confira a aba `Por professor`.

### 6.4 Editar uma solicitacao existente

1. Selecione a data no calendario.
2. Abra a solicitacao desejada.
3. Ajuste os campos necessarios no formulario lateral.
4. Clique novamente em `Salvar solicitacao`.

Use o botao `Nova` quando quiser limpar o formulario e iniciar um novo cadastro na data selecionada.

### 6.5 Excluir uma solicitacao

O botao `Excluir` remove a solicitacao selecionada.

Atencao:

- se ja existirem arquivos enviados para essa solicitacao, a exclusao pode ser bloqueada;
- antes de excluir, confirme se a data e o documento selecionados sao realmente os corretos.

### 6.6 Acompanhar os professores

Na aba `Por professor`, a gestao consegue:

- ver quantos professores ja enviaram;
- identificar quem ainda esta pendente;
- abrir os detalhes por professor;
- conferir disciplina, turma e horario vinculado quando aplicavel.

Essa visualizacao e a melhor opcao para acompanhar pendencias.

### 6.7 Consultar os arquivos enviados

Na aba `Arquivos`, a gestao consegue:

- ver os anexos agrupados por professor;
- conferir o nome informado pelo usuario;
- identificar o nome padronizado salvo no sistema, quando diferente;
- baixar o arquivo enviado.

Essa aba e a melhor opcao para conferencia final dos materiais recebidos.

## 7. Boas praticas para professores

- Nomeie os arquivos de forma clara antes do envio.
- Confira a disciplina e a turma antes de anexar.
- Nao deixe o envio para muito perto do prazo final.
- Se precisar substituir um arquivo, remova e reenvi e somente enquanto o prazo estiver aberto.
- Revise se o arquivo correto aparece como enviado logo apos a confirmacao.

## 8. Boas praticas para a gestao

- Cadastre titulos objetivos no campo `Documento solicitado`.
- Use observacoes curtas e claras sobre o que deve ser entregue.
- Verifique o publico-alvo antes de divulgar a solicitacao.
- Acompanhe pendencias pela aba `Por professor`.
- Use a aba `Arquivos` para validar rapidamente os anexos ja recebidos.

## 9. Dificuldades comuns

### 9.1 A entrega nao aparece para o professor

Verifique se:

- a data correta foi selecionada;
- o ano letivo esta certo;
- a solicitacao foi criada para o publico esperado;
- o professor esta vinculado corretamente a turma e disciplina.

### 9.2 O professor enviou o arquivo errado

Se o prazo ainda estiver aberto:

1. remover o arquivo atual;
2. reenviar a versao correta.

Se o prazo ja estiver encerrado, a equipe gestora deve avaliar o procedimento interno da escola.

### 9.3 O nome exibido nao e igual ao nome final do download

Isso pode acontecer de forma esperada.

O sistema mostra:

- o nome reconhecivel para o usuario;
- e, quando necessario, o nome padronizado usado internamente.

No download, prevalece o nome padronizado do sistema.

## 10. Resumo rapido

- `Professor`: consulta pendencias, envia, baixa e substitui arquivos dentro do prazo.
- `Gestao`: cadastra solicitacoes, acompanha professores e baixa anexos.
- `Prazo aberto`: permite envio e substituicao.
- `Prazo encerrado`: permite apenas consulta do que ja foi enviado.
- `Nome do usuario` e `nome do sistema` podem aparecer juntos para evitar confusao.
